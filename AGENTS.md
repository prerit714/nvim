# AGENTS.md

## Repo Shape
- This is a personal Neovim config, not a package project; there is no package manager manifest, CI workflow, or test suite in the repo.
- `init.lua` is the entrypoint. It sets leaders and Neovide zoom keys, then requires `options`, `keymaps`, `lazy-bootstrap`, and `lazy-plugins` in that order.
- Plugin specs live in `lua/kickstart/plugins/*.lua` and `lua/custom/plugins/*.lua`; `lua/lazy-plugins.lua` imports both explicit kickstart specs and `{ import = "custom.plugins" }`.
- `lazy-lock.json` is generated by lazy.nvim and pins plugin commits. Do not edit it by hand; let lazy.nvim update it.

## Commands
- Smoke-check startup with `nvim --headless -u init.lua +qa`. This may bootstrap lazy.nvim/plugins into Neovim's data dir on a fresh machine.
- Format Lua with `stylua .` using `.stylua.toml` if StyLua is installed on the shell PATH. In this setup, StyLua may only be available through Mason inside Neovim.
- There is no repo-local lint/typecheck/test command to run outside Neovim.

## Style And Config Conventions
- Lua formatting is 2 spaces, 80 columns, Unix line endings, double quotes preferred when StyLua chooses, and simple statements may collapse onto one line.
- Keep plugin additions as lazy.nvim specs; prefer `lua/custom/plugins/<name>.lua` for local additions rather than editing kickstart-derived files unless changing shared core behavior.
- `vim.g.have_nerd_font` defaults to `false`; preserve ASCII/non-Nerd-Font fallbacks when adding UI symbols.
- `options.lua` contains both editor options and custom statusline/timer logic, including `:TimerStart <1-60>`.

## Tooling Gotchas
- LSP/Mason installation is configured in `lua/kickstart/plugins/lspconfig.lua`; configured servers/tools include `clangd`, `cmake`, `gopls`, `pyrefly`, `rust_analyzer`, `lua_ls`, `glsl_analyzer`, and `stylua`.
- Formatting on save is via conform.nvim and currently only maps Lua to `stylua`; C/C++ format-on-save is explicitly disabled.
- nvim-lint is loaded, but all configured default linters are immediately disabled, so do not expect lint diagnostics from it without changing that file.
- Treesitter uses the `main` branch API and calls `require("nvim-treesitter").install(parsers)` rather than the older `ensure_installed` setup style.
